Data Visualization Planning

Static Data

I intended to showcase how data was measured with the Coronal Mass Ejections by using a speed graph would show a relationship between speed and other data. I decided to use Time and the Half Angle as I believed that it would offer a better understanding to viewers as a speed graph is part of the basic understanding of Physical Sciences. I played around with which graph would better display the information needed and I made sure that what I chose would show sufficient data. This is why I chose a scatter plot for Speed vs. Time and a line graph for the Speed vs. Half-Angle graph.

Interactive Data

I decided against using a sophisticated data visualization tool and focused more on the fact that the line graph did not transfer the data's information well. I decided to create an interactive line graph. The interactivity comes from the tooltips that users will find when they hover over the circles at the beginning and end of the line.

Data Art

For the creation of this data art, I determined the necessary information for the chart, including the event date, CME speed, half-angle, and catalog. Next, I sketched out a basic layout for the chart, taking into account the use of D3.js for data visualization and interaction. To ensure organization and flexibility, I opted to create a modular and reusable function, aptly named "createTimelineChart". This function was designed to take in various parameters, such as the target selector, data, and configuration options. Throughout the planning process, I paid close attention to potential errors, implementing measures to handle them, such as checking for empty datasets and properly parsing dates. Finally, I addressed styling concerns by adding CSS rules to create a visually pleasing and cohesive presentation of the timeline chart.

For the Exam Data Art I carefully identified the key data attributes, such as type, speed, time, latitude, longitude, half angle, and note. After much consideration, I decided to use a Circular Packing chart because of its ability to display hierarchical data in a circular layout, providing a unique and visually appealing perspective. The code implementation involved processing the API data, formatting it to fit the Circular Packing structure, and setting up the necessary scales and elements. I also added labels and tooltips to enhance interactivity and provide comprehensive information about each data point. Intended to randomise the position and colour of the circles for a more engaging experience
